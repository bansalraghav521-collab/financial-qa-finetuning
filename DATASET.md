# DATASET.md — Dataset Design & Preparation

## Dataset Choice

**Primary Dataset:** `finance-alpaca` on HuggingFace  
**Link:** https://huggingface.co/datasets/gbharti/finance-alpaca  
**License:** Apache 2.0 — free for research use  
**Original size:** ~68,000 samples  
**Subset used:** 1,500 curated samples  

---

## Why This Dataset

- Specifically designed for financial instruction tuning — questions and answers are grounded in real financial concepts
- Already in Alpaca instruction format (instruction / input / output), so formatting overhead is minimal
- Large enough to filter down to a high-quality subset without running out of data
- Community-validated: widely used as a starting point for finance LLM fine-tuning

---

## Data Cleaning Steps

### Step 1 — Filter for Quality

Remove samples where:
- `output` field is fewer than 20 words (too short to be meaningful)
- `output` field is over 300 words (too long for SLM generation targets)
- `instruction` is a duplicate (exact string match)
- Answer contains obvious formatting artifacts (broken HTML, escape sequences)

```python
def is_valid_sample(sample):
    output_words = len(sample["output"].split())
    return (
        20 <= output_words <= 300
        and len(sample["instruction"].strip()) > 10
        and "<html" not in sample["output"].lower()
    )
```

### Step 2 — Deduplicate

Use fuzzy matching (sequence ratio > 0.85) to catch near-duplicate instructions. Keeps the version with the longer, more detailed output.

### Step 3 — Category Balancing

Ensure the 1,500-sample subset is balanced across core financial topics:

| Topic | Target Samples |
|---|---|
| Interest rates & bonds | ~250 |
| Stock market & equities | ~250 |
| Personal finance & budgeting | ~200 |
| Economic indicators (GDP, CPI) | ~200 |
| Financial instruments (ETFs, options) | ~200 |
| Tax concepts | ~150 |
| Risk & portfolio theory | ~150 | 
| **Total** | **1,500** |

### Step 4 — Format to Instruction Tuning Style

Every sample is converted to a consistent prompt template:

```
### Instruction:
{instruction}

### Input:
{input}

### Response:
{output}
```

If `input` is empty (most samples), it is omitted from the prompt.

**Example formatted sample:**
```
### Instruction:
Explain the difference between a Roth IRA and a Traditional IRA.

### Response:
A Traditional IRA allows pre-tax contributions, meaning you get a tax deduction now but pay taxes on withdrawals in retirement. A Roth IRA uses after-tax contributions — no deduction now, but qualified withdrawals in retirement are tax-free. The key deciding factor is whether you expect your tax rate to be higher now or in retirement.
```

## Train / Val / Eval Split

| Split | Size | Purpose |
|---|---|---|
| `train.jsonl` | 1,200 samples (80%) | Model training |
| `val.jsonl` | 150 samples (10%) | Loss monitoring during training (early stopping) |
| `eval.jsonl` | 150 samples (10%) | Final held-out evaluation — never seen during training |

The eval set is **stratified** — each topic category is proportionally represented. This ensures ROUGE scores are not inflated by easy questions or deflated by unusually hard ones.

## Bonus: Synthetic Data Augmentation (Optional)

To further enrich the training set, a small number of synthetic samples (~50) were generated by:
1. Taking existing questions and rephrasing the instruction differently
2. Writing alternative correct answers grounded in the same financial fact

This simulates prompt diversity and helps the model generalize to different phrasings of the same financial concept.

> Note: Synthetic samples are clearly flagged with `"synthetic": true` in the JSONL and are excluded from the evaluation set.

## Reproducibility

To reproduce the dataset:

```bash
python src/prepare_data.py \
  --source gbharti/finance-alpaca \
  --n_samples 1500 \
  --output_dir ./data/ \
  --seed 42
```

Setting `--seed 42` ensures the same train/val/eval split every time.
